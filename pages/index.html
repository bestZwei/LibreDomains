申请指南

1

Fork 仓库

访问 GitHub 仓库 并 Fork 到你的账户

2

创建配置文件

在 domains/ciao.su/ 目录下创建 JSON 配置文件

3

填写配置

按照指定格式填写域名配置信息

4

提交 PR

提交 Pull Request 并等待自动检查和审核

5

域名生效

PR 合并后，域名记录将在几分钟内生效

配置文件示例

{
  "owner": {
    "name": "你的名字",
    "github": "你的GitHub用户名",
    "email": "你的邮箱"
  },
  "records": [
    {
      "type": "A",
      "name": "@",
      "content": "185.199.108.153",
      "ttl": 3600,
      "proxied": true
    },
    {
      "type": "CNAME",
      "name": "www",
      "content": "你的GitHub用户名.github.io",
      "ttl": 3600,
      "proxied": true
    }
  ]
}

申请规则

域名格式

  • 小写字母、数字和连字符组成
  • 长度在 2-63 个字符之间
  • 不能以连字符开头或结尾

数量限制

  • 每个用户最多申请 3 个子域名
  • 每个子域名最多 10 个 DNS 记录

保留域名

  • 系统保留：www, mail, api, admin
  • 服务保留：cdn, ftp, ns, dns
  • 开发保留：dev, test, staging
